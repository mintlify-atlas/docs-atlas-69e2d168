## Overview

The Products and Services features allow Okulink Safaris to offer supplementary items and services that enhance the safari experience. From safari gear and equipment to specialized services like photography packages and transport arrangements, these features provide additional revenue streams and customer convenience.

<Note>
Products and Services are separate features with similar functionality but distinct purposes. Products are physical items available for purchase, while Services are intangible offerings provided to customers.
</Note>

### Product Information

#### Database Fields

| Field | Type | Description |
|-------|------|-------------|
| `name` | string | Product name |
| `slug` | string | URL-friendly identifier (unique) |
| `category` | string | Product category or type |
| `location` | string | Where product is available/sourced |
| `price` | string | Product price |
| `quantity_available` | integer | Current stock level |
| `description` | text | Detailed product information |
| `image` - `image6` | string | Up to 6 product images |
| `user_id` | integer | Administrator who created the listing |
| `published` | integer | Publication status (0 = draft, 1 = published) |

### Creating a Product

<Steps>
  <Step title="Access Product Creation">
    Navigate to `/product/new` (route name: `products.create`) from the admin dashboard
    
    <Info>Requires authentication and email verification</Info>
  </Step>
  
  <Step title="Enter Product Details">
    Complete all required fields:
    
    - **Name**: Product name (e.g., "Professional Binoculars 10x42")
    - **Category**: Product type (e.g., "Safari Equipment", "Clothing", "Photography Gear")
    - **Location**: Availability location or source
    - **Price**: Product cost
    - **Quantity Available**: Current stock level
    - **Description**: Comprehensive product details, specifications, and benefits
  </Step>
  
  <Step title="Upload Product Images">
    - **Main Image**: Primary product photo (required)
    - **Second Image**: Alternative view or detail shot (required)
    - **Additional Images**: Up to 4 more images showing different angles, features, or uses (optional)
  </Step>
  
  <Step title="Create Product">
    Submit to save the product. The system will:
    - Generate a slug from the product name
    - Store all images
    - Create product in draft mode
    - Associate with your user account
    - Redirect to products management page
  </Step>
</Steps>

<Steps>
  <Step title="Navigate to Edit">
    Access `/product/edit/{id}` from the products management page
  </Step>
  
  <Step title="Update Information">

</Steps>

## Best Practices

<AccordionGroup>
  <Accordion title="Product Management">
    **Inventory Control:**
    - Keep `quantity_available` updated
    - Set to 0 when out of stock (or unpublish)
    - Update after each sale
    - Plan restock levels
    
    **Product Descriptions:**
    - Include specifications and features
    - Mention brand names if applicable
    - List what's included
    - Note any requirements or compatibility
    - Include care instructions if relevant
    
    **Pricing:**
    - Research competitive pricing
    - Include any taxes or fees
    - Offer bundle discounts
    - Update seasonal pricing
  </Accordion>
  
  <Accordion title="Service Management">
    **Service Descriptions:**
    - Clearly state what's included
    - Specify duration
    - List requirements or prerequisites
    - Mention any limitations or restrictions
    - Include booking/cancellation terms
    
    **Pricing Structure:**
    - Be transparent about what's covered
    - Offer tiered packages when applicable
    - Note any additional costs
    - Include seasonal variations
    
    **Service Delivery:**
    - Set realistic expectations
    - Define service areas/locations
    - Specify availability (days, times)
    - Include contact information for booking
  </Accordion>
  
  <Accordion title="Image Strategy">
    **For Products:**
    - Clean, well-lit product photos
    - Multiple angles and views
    - Show scale/size reference
    - Detail shots of key features
    - In-use context photos
    
    **For Services:**
    - Show service in action
    - Include staff/guides providing service
    - Display results or outcomes
    - Showcase various service aspects
    - Use authentic customer photos
  </Accordion>
</AccordionGroup>

## Inventory Management (Products)

<Warning>
The current system tracks quantity but doesn't automatically decrement on purchase. You'll need to manually update quantities after sales or implement automatic inventory management.
</Warning>

**Recommended Workflow:**
1. Set initial `quantity_available` when creating product
2. Monitor sales through your order system
3. Manually update quantities regularly
4. Set up low stock alerts
5. Unpublish or mark as "out of stock" when quantity reaches 0
6. Republish when restocked

## Troubleshooting

<AccordionGroup>
  <Accordion title="Product Not Selling">
    **Potential Issues:**
    - Check that product is published
    - Verify quantity_available is greater than 0
    - Ensure price is competitive
    - Review product description for completeness
    - Check image quality and variety
    - Confirm category is correct
  </Accordion>
  
  <Accordion title="Service Booking Confusion">
    **Solutions:**
    - Clarify booking process in description
    - Include clear call-to-action
    - Provide contact information
    - Specify availability clearly
    - Set clear expectations
  </Accordion>
  
  <Accordion title="Category Filter Not Working">
    **Check:**
    - Category name spelling and capitalization
    - No extra spaces in category field
    - Item is published
    - Cache cleared if caching enabled
  </Accordion>
</AccordionGroup>
