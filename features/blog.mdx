## Overview

The Blog feature enables Okulink Safaris to create compelling content that attracts potential customers, shares safari experiences, provides travel tips, and establishes expertise in safari tourism. The blog supports rich media content with multiple images and is integrated throughout the platform to drive engagement.

## Blog Post Information

### Database Fields

Each blog post contains:

| Field | Type | Description |
|-------|------|-------------|
| `title` | text | Blog post title/headline |
| `content` | text | Full article content (supports HTML) |
| `image` | text | Main featured image path |
| `image2` - `image6` | text | Additional images for the post |
| `user_id` | integer | Author (administrator) ID |
| `published` | integer | Publication status (0 = draft, 1 = published) |

<Note>
The blog model has basic fillable fields (image, title, content) with additional image fields added via migrations.
</Note>

## Creating a Blog Post

<Steps>
  <Step title="Access Blog Creation">
    Navigate to `/blogs/create` (route name: `blogs.create`) from the admin dashboard
    
    <Info>Requires authentication and email verification</Info>
  </Step>
  
  <Step title="Write Your Content">
    **Title**: Create an engaging, descriptive headline
    - Keep it concise but informative
    - Include keywords for SEO
    - Make it compelling to encourage clicks
    
    **Content**: Write your full article
    - Use clear paragraphs and sections
    - Include relevant information and details
    - Add formatting for readability
    - Incorporate storytelling elements
    - Include practical tips and advice
  </Step>
  
  <Step title="Add Images">
    Upload visual content to enhance your post:
    
    - **Main Image**: Featured image (required, JPG/PNG/SVG/WebP)
    - **Second Image**: Secondary image (required)
    - **Additional Images**: Up to 4 more images (optional, image3-6)
    
    Use images that:
    - Support your written content
    - Show safari experiences
    - Capture wildlife and landscapes
    - Illustrate tips or advice
    - Break up long text sections
  </Step>
  
  <Step title="Save Blog Post">
    Click "Submit" to create the post. The system will:
    - Store all content and images
    - Create post in draft mode (unpublished)
    - Associate post with your user account
    - Redirect to blogs management page
    - Show success message: "Content Created Successfully!"
  </Step>
</Steps>

## Managing Blog Posts

### Viewing Blog Posts

<Tabs>
  <Tab title="Admin View">
    Access all blog posts at `/all-blogs` (route name: `all-blogs`):
    
    **Features:**
    - All blog posts (published and drafts)
    - 10 posts per page with pagination
    - Most recent posts first
    - Quick action buttons (edit, publish, delete)
    - Current publication status visible
    
    <Note>Alternative admin route: `/blogs` (route name: `blogs`) also shows admin management</Note>
  </Tab>
  
  <Tab title="Public Blog Page">
    Customers view blogs at `/blog` (route name: `blog`):
    
    **Features:**
    - Only published posts displayed
    - 6 cover/featured posts with pagination
    - Latest 2 posts highlighted in sidebar
    - Engaging layout for reading
  </Tab>
</Tabs>

### Reading a Blog Post

Individual posts are accessible at `/blogs/show/{id}` (route name: `blogs.show`):

**Post Page Includes:**
- Full article title and content
- All post images in gallery format
- Latest 2 blog posts in sidebar for continued engagement
- Latest 4 posts at bottom for related reading
- Publication date and metadata

### Editing a Blog Post

<Steps>
  <Step title="Navigate to Edit Page">
    From the blogs management page, click "Edit" or visit `/blogs/edit/{id}`
  </Step>
  
  <Step title="Modify Content">
    Update any aspect of the post:
    - Title
    - Content/article text
    - Can add, remove, or modify any section
  </Step>
  
  <Step title="Update Images (Optional)">
    Replace or add images:
    - If you upload any new images, all image fields are processed
    - If no images are uploaded, existing images are preserved
    - Can selectively update specific images
  </Step>
  
  <Step title="Save Changes">
    Click "Update" to save your edits
    - User association updated to current user
    - Redirected to admin blogs page
    - Success message: "Content updated Successfully!"
  </Step>
</Steps>

### Publishing and Unpublishing

<Tabs>
  <Tab title="Publishing a Draft">
    To make a blog post visible to readers:
    
    1. Navigate to `/all-blogs` or `/blogs`
    2. Find the post with published status 0 (draft)
    3. Click "Publish" (PUT request to `/blogs/publish/{id}`)
    4. Status changes to 1 (published)
    5. Post now appears on public blog pages
    6. Post included in sidebar and featured sections
  </Tab>
  
  <Tab title="Unpublishing a Post">
    To remove a post from public view:
    
    1. Navigate to the blogs management page
    2. Find the post with published status 1
    3. Click "Unpublish"
    4. Status changes to 0 (draft)
    5. Post hidden from public pages
    6. Post still accessible in admin view
  </Tab>
</Tabs>

<Info>
Use draft mode to prepare content in advance, review posts before publication, or temporarily hide outdated content without deleting it.
</Info>

### Deleting a Blog Post

<Warning>
Deleting a blog post permanently removes it from the database. This action cannot be undone. Consider unpublishing instead if the content might be useful later.
</Warning>

To delete a post:
1. Navigate to blogs management page
2. Click "Delete" on the post
3. Confirm deletion
4. Post permanently removed with message: "Content Deleted"

## Routes Reference

| Action | Method | Route | Route Name | Auth Required |
|--------|--------|-------|------------|---------------|
| List all (admin) | GET | `/all-blogs` | `all-blogs` | No* |
| Admin management | GET | `/blogs` | `blogs` | No* |
| Public blog page | GET | `/blog` | `blog` | No |
| Create form | GET | `/blogs/create` | `blogs.create` | Yes (verified) |
| Store new post | POST | `/blogs/store` | `blogs.store` | Yes (verified) |
| Show post | GET | `/blogs/show/{id}` | `blogs.show` | No |
| Edit form | GET | `/blogs/edit/{id}` | `blogs.edit` | Yes (verified) |
| Update post | POST | `/blogs/update/{id}` | `blogs.update` | Yes (verified) |
| Delete post | DELETE | `/blogs/delete/{id}` | `blogs.delete` | Yes (verified) |
| Toggle publish | PUT | `/blogs/publish/{id}` | `blogs.publish` | Yes (verified) |

<Note>
*While admin blog list routes don't enforce authentication at the route level, creation, editing, and deletion require authentication and verification.
</Note>

## Writing Effective Blog Posts

<Tabs>
  <Tab title="Structure">
    **Recommended Post Structure:**
    
    1. **Engaging Opening** (1-2 paragraphs)
       - Hook the reader immediately
       - State what the post will cover
       - Why it matters to readers
    
    2. **Main Content** (Multiple sections)
       - Use clear headings
       - Break into digestible chunks
       - Include specific examples
       - Add practical tips
       - Support with images
    
    3. **Conclusion** (1 paragraph)
       - Summarize key points
       - Call to action
       - Encourage engagement
    
    4. **Call to Action**
       - Book a tour
       - Contact for more information
       - Share the post
       - Read related content
  </Tab>
  
  <Tab title="Writing Tips">
    **Do:**
    - Write conversationally but professionally
    - Use active voice
    - Include personal experiences
    - Add specific details and examples
    - Break up long paragraphs
    - Use bullet points and lists
    - Proofread carefully
    - Include actionable advice
    
    **Don't:**
    - Use jargon without explanation
    - Write overly long paragraphs
    - Forget to proofread
    - Neglect image captions
    - Overlook mobile readability
    - Skip the conclusion
    - Forget calls to action
  </Tab>
  
  <Tab title="Engagement">
    **Encourage Interaction:**
    - Ask questions in the content
    - Invite comments and feedback
    - Include social sharing buttons
    - Link to related tours or services
    - Offer downloadable resources
    - Include contact information
    
    **Build Authority:**
    - Cite sources when using statistics
    - Include expert quotes
    - Share insider knowledge
    - Demonstrate expertise
    - Provide unique insights
  </Tab>
</Tabs>

## Troubleshooting

<AccordionGroup>
  <Accordion title="Images Not Displaying in Post">
    **Problem**: Images uploaded but not showing in published post
    
    **Solutions**:
    - Verify storage symlink exists: `php artisan storage:link`
    - Check image file formats (JPG, PNG, SVG, WebP only)
    - Confirm file permissions on storage directory
    - Clear browser cache
    - Check that images were successfully uploaded
    - Verify image paths in database
  </Accordion>
  
  <Accordion title="Post Not Appearing on Public Page">
    **Problem**: Published post doesn't show on blog page
    
    **Solutions**:
    - Verify published status is 1 (published, not 0)
    - Clear application cache if caching enabled
    - Check that title and content are not empty
    - Refresh the blog page
    - Verify post ID exists in database
  </Accordion>
  
  <Accordion title="Content Formatting Issues">
    **Problem**: Content doesn't display as intended
    
    **Solutions**:
    - Check HTML formatting in content field
    - Verify no unclosed tags
    - Test in different browsers
    - Review CSS conflicts
    - Use simple, clean HTML
  </Accordion>
  
  <Accordion title="Cannot Edit Published Post">
    **Problem**: Edit page doesn't load or save
    
    **Solutions**:
    - Verify authentication and email verification
    - Check post ID is correct
    - Ensure required fields are filled
    - Clear browser cache
    - Try different browser
  </Accordion>
</AccordionGroup>
