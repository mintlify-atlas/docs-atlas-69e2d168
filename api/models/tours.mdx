## Overview

The `Tours` model represents safari tour packages in the Okulink Safaris system. It manages tour details including accommodation, transport, pricing, images, and categorization.

## Database Table

**Table Name:** `tours`

## Fillable Fields

The following fields can be mass-assigned:

- `hotel`
- `transport`
- `image`
- `per_person_sharing`
- `single_room`
- `meals`

## Complete Schema

<ResponseField name="id" type="integer" required>
  Primary key identifier for the tour
</ResponseField>

<ResponseField name="hotel" type="string" required>
  Name of the hotel or accommodation included in the tour package
</ResponseField>

<ResponseField name="transport" type="text" required>
  Description of transport arrangements for the tour
</ResponseField>

<ResponseField name="location" type="text" required>
  Geographic location or destination of the tour
</ResponseField>

<ResponseField name="category" type="text" required>
  Main category classification for the tour (e.g., Safari, Beach, Mountain)
</ResponseField>

<ResponseField name="sub_category" type="text" required>
  Sub-category for more specific tour classification
</ResponseField>

<ResponseField name="image" type="text" required>
  Primary image URL for the tour
</ResponseField>

<ResponseField name="image1" type="text" required>
  Secondary image URL for the tour gallery
</ResponseField>

<ResponseField name="image2" type="text" required>
  Third image URL for the tour gallery
</ResponseField>

<ResponseField name="image4" type="string" required>
  Fourth image URL for the tour gallery
</ResponseField>

<ResponseField name="image5" type="string" required>
  Fifth image URL for the tour gallery
</ResponseField>

<ResponseField name="image6" type="string" required>
  Sixth image URL for the tour gallery
</ResponseField>

<ResponseField name="per_person_sharing" type="text" required>
  Price per person when sharing accommodation
</ResponseField>

<ResponseField name="single_room" type="text" required>
  Price for single room occupancy
</ResponseField>

<ResponseField name="meals" type="text" required>
  Description of meal arrangements included in the tour
</ResponseField>

<ResponseField name="additional_info" type="text">
  Additional information or special notes about the tour (nullable)
</ResponseField>

<ResponseField name="published" type="integer" default="0">
  Publication status of the tour (0 = draft, 1 = published)
</ResponseField>

<ResponseField name="user_id" type="integer" required>
  ID of the user who created the tour
</ResponseField>

<ResponseField name="created_at" type="timestamp">
  Timestamp when the tour was created
</ResponseField>

<ResponseField name="updated_at" type="timestamp">
  Timestamp when the tour was last updated
</ResponseField>

## Relationships

### User

Each tour belongs to a user (author/creator) through the `user_id` foreign key.
