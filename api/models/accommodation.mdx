---
title: Accommodation model
description: Database schema and structure for accommodations
---

The Accommodation model represents lodging options including hotels, lodges, and camps.

## Table Structure

The `accomodations` table (note the spelling in the database) stores accommodation listings.

### Fields

<ResponseField name="id" type="integer">
  Unique identifier for the accommodation
</ResponseField>

<ResponseField name="name" type="string">
  Accommodation name
</ResponseField>

<ResponseField name="description" type="text">
  Detailed accommodation description
</ResponseField>

<ResponseField name="location" type="string">
  Accommodation location
</ResponseField>

<ResponseField name="category" type="string">
  Accommodation category (hotel, lodge, camp, etc.)
</ResponseField>

<ResponseField name="rates" type="text">
  Pricing information and rates
</ResponseField>

<ResponseField name="slug" type="string">
  URL-friendly identifier for the accommodation
</ResponseField>

<ResponseField name="image" type="text">
  Primary image URL
</ResponseField>

<ResponseField name="image2" type="string">
  Second image URL (optional)
</ResponseField>

<ResponseField name="image3" type="string">
  Third image URL (optional)
</ResponseField>

<ResponseField name="image4" type="string">
  Fourth image URL (optional)
</ResponseField>

<ResponseField name="image5" type="string">
  Fifth image URL (optional)
</ResponseField>

<ResponseField name="image6" type="string">
  Sixth image URL (optional)
</ResponseField>

<ResponseField name="user_id" type="integer">
  ID of the user who created the accommodation listing
</ResponseField>

<ResponseField name="published" type="integer">
  Publication status (0 = unpublished, 1 = published)
</ResponseField>

<ResponseField name="created_at" type="timestamp">
  Timestamp when the accommodation was created
</ResponseField>

<ResponseField name="updated_at" type="timestamp">
  Timestamp when the accommodation was last updated
</ResponseField>

## Fillable Fields

The following fields are mass-assignable:

- `name`
- `location`
- `description`
- `category`
- `rates`
