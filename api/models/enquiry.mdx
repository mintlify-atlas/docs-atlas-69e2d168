---
title: Enquiry model
description: Database schema and structure for customer enquiries
---

The Enquiry model represents customer trip planning enquiries and information requests.

## Table Structure

The `enquiries` table stores customer enquiry data for trip planning and booking requests.

### Fields

<ResponseField name="id" type="integer">
  Unique identifier for the enquiry
</ResponseField>

<ResponseField name="full_name" type="text">
  Customer's full name
</ResponseField>

<ResponseField name="email_address" type="text">
  Customer's email address
</ResponseField>

<ResponseField name="phone_number" type="text">
  Customer's phone number
</ResponseField>

<ResponseField name="destinations" type="text">
  Desired destinations for the trip
</ResponseField>

<ResponseField name="nationality" type="text">
  Customer's nationality
</ResponseField>

<ResponseField name="travel_date" type="text">
  Preferred travel date
</ResponseField>

<ResponseField name="adults" type="text">
  Number of adults traveling
</ResponseField>

<ResponseField name="kids" type="text">
  Number of children traveling
</ResponseField>

<ResponseField name="budget" type="text">
  Customer's budget range
</ResponseField>

<ResponseField name="medium" type="text">
  Medium accommodation preference (yes/no)
</ResponseField>

<ResponseField name="luxury" type="text">
  Luxury accommodation preference (yes/no)
</ResponseField>

<ResponseField name="additional_info" type="text" nullable>
  Additional information or special requests
</ResponseField>

<ResponseField name="created_at" type="timestamp">
  Timestamp when the enquiry was submitted
</ResponseField>

<ResponseField name="updated_at" type="timestamp">
  Timestamp when the enquiry was last updated
</ResponseField>

## Fillable Fields

The following fields are mass-assignable:

- `full_name`
- `email_address`
- `phone_number`
- `destinations`
- `nationality`
- `travel_date`
