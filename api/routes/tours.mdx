## Overview

The Tours API provides endpoints for browsing, creating, updating, and managing safari tour packages. Tours can be categorized and include details about accommodations, transport, pricing, and images.

## Browse All Safaris

<api method="GET" path="/safaris" />

Retrieve a paginated list of all available safari tours in random order.

### Response

<ResponseField name="safaris" type="array">
  Paginated collection of safari tours (20 per page)
  
  <Expandable title="Tour Object">
    <ResponseField name="id" type="integer">
      Unique tour identifier
    </ResponseField>
    
    <ResponseField name="hotel" type="string">
      Hotel/accommodation name
    </ResponseField>
    
    <ResponseField name="transport" type="string">
      Transportation details
    </ResponseField>
    
    <ResponseField name="single_room" type="decimal">
      Price for single room occupancy
    </ResponseField>
    
    <ResponseField name="per_person_sharing" type="decimal">
      Price per person for shared occupancy
    </ResponseField>
    
    <ResponseField name="meals" type="string">
      Meal plan details
    </ResponseField>
    
    <ResponseField name="category" type="string">
      Tour category
    </ResponseField>
    
    <ResponseField name="sub_category" type="string">
      Tour subcategory
    </ResponseField>
    
    <ResponseField name="location" type="string">
      Tour location/destination
    </ResponseField>
    
    <ResponseField name="additional_info" type="text">
      Additional tour information
    </ResponseField>
    
    <ResponseField name="image" type="string">
      Main tour image path
    </ResponseField>
    
    <ResponseField name="image1" type="string">
      Secondary tour image path
    </ResponseField>
    
    <ResponseField name="image2" type="string" optional>
      Additional tour image path
    </ResponseField>
    
    <ResponseField name="image3" type="string" optional>
      Additional tour image path
    </ResponseField>
    
    <ResponseField name="image4" type="string" optional>
      Additional tour image path
    </ResponseField>
    
    <ResponseField name="image5" type="string" optional>
      Additional tour image path
    </ResponseField>
    
    <ResponseField name="image6" type="string" optional>
      Additional tour image path
    </ResponseField>
    
    <ResponseField name="published" type="boolean">
      Publication status (0 or 1)
    </ResponseField>

      ID of user who created the tour
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="categories" type="array">
  All available tour categories
</ResponseField>

<ResponseField name="blogs" type="array">
  Latest 2 blog posts

### Example Request

```bash
curl https://yourdomain.com/safaris
```

## List All Tours (Admin)

<api method="GET" path="/addtour" />

Retrieve a paginated list of all tours for admin management.

### Authentication

<Note>
  Requires authentication and verified email
</Note>

**Middleware:** `auth`, `verified`

### Response

Returns paginated list of tours (20 per page) ordered by most recent.

### Example Request

```bash
curl https://yourdomain.com/addtour \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Cookie: laravel_session=YOUR_SESSION"
```

## Create Tour (Admin)

<api method="POST" path="/addtour/store" />

Create a new safari tour.

### Authentication

<Note>
  Requires authentication and verified email
</Note>

**Middleware:** `auth`, `verified`

### Request Body

<ParamField body="hotel" type="string" required>
  Hotel or accommodation name
</ParamField>

<ParamField body="transport" type="string" required>
  Transportation details
</ParamField>

<ParamField body="single_room" type="decimal" required>
  Price for single room occupancy
</ParamField>

<ParamField body="per_person_sharing" type="decimal" required>
  Price per person for shared occupancy
</ParamField>

<ParamField body="meals" type="string" required>
  Meal plan details
</ParamField>

<ParamField body="category" type="string" required>
  Tour category
</ParamField>

<ParamField body="sub_category" type="string" required>
  Tour subcategory
</ParamField>

<ParamField body="location" type="string" required>
  Tour location/destination
</ParamField>

<ParamField body="additional_info" type="text" optional>
  Additional information about the tour
</ParamField>

<ParamField body="image" type="file" required>
  Main tour image (jpg, jpeg, svg, png, webp)
</ParamField>

<ParamField body="image1" type="file" required>
  Secondary tour image (jpg, jpeg, svg, png, webp)
</ParamField>

<ParamField body="image2" type="file" optional>
  Additional tour image (jpg, jpeg, svg, png, webp)
</ParamField>

<ParamField body="image3" type="file" optional>
  Additional tour image (jpg, jpeg, svg, png, webp)
</ParamField>

<ParamField body="image4" type="file" optional>
  Additional tour image (jpg, jpeg, svg, png, webp)
</ParamField>

<ParamField body="image5" type="file" optional>
  Additional tour image (jpg, jpeg, svg, png, webp)
</ParamField>

<ParamField body="image6" type="file" optional>
  Additional tour image (jpg, jpeg, svg, png, webp)
</ParamField>

### Example Request

```bash
curl -X POST https://yourdomain.com/addtour/store \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Content-Type: multipart/form-data" \
  -F "hotel=Serena Safari Lodge" \
  -F "transport=4x4 Safari Vehicle" \
  -F "single_room=1500.00" \
  -F "per_person_sharing=900.00" \
  -F "meals=Full Board" \
  -F "category=Wildlife Safari" \
  -F "sub_category=Luxury Safari" \
  -F "location=Maasai Mara, Kenya" \
  -F "additional_info=Experience the great migration" \
  -F "image=@/path/to/image.jpg" \
  -F "image1=@/path/to/image1.jpg"
```

### Response

Redirects to tours list with success message: "You Added a Tour."

## Update Tour (Admin)

<api method="POST" path="/addtour/update/{id}" />

Update an existing safari tour.

### Authentication

<Note>
  Requires authentication and verified email
</Note>

**Middleware:** `auth`, `verified`

### Path Parameters

<ParamField path="id" type="integer" required>
  The unique tour ID
</ParamField>

### Request Body

<ParamField body="hotel" type="string" required>
  Hotel or accommodation name
</ParamField>

<ParamField body="transport" type="string" required>
  Transportation details
</ParamField>

<ParamField body="single_room" type="decimal" required>
  Price for single room occupancy
</ParamField>

<ParamField body="per_person_sharing" type="decimal" required>
  Price per person for shared occupancy
</ParamField>

<ParamField body="meals" type="string" required>
  Meal plan details
</ParamField>

<ParamField body="category" type="string" required>
  Tour category
</ParamField>

<ParamField body="sub_category" type="string" required>
  Tour subcategory
</ParamField>

<ParamField body="additional_info" type="text" optional>
  Additional information about the tour
</ParamField>

<ParamField body="image" type="file" optional>
  Main tour image (only if updating)
</ParamField>

<ParamField body="image2" type="file" optional>
  Additional tour images (only if updating)
</ParamField>

<Note>
  Images are optional during update. If not provided, existing images are retained.
</Note>

### Example Request

```bash
curl -X POST https://yourdomain.com/addtour/update/123 \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Content-Type: multipart/form-data" \
  -F "hotel=Updated Serena Safari Lodge" \
  -F "transport=Luxury 4x4 Safari Vehicle" \
  -F "single_room=1800.00" \
  -F "per_person_sharing=1100.00" \
  -F "meals=Full Board with Premium Drinks" \
  -F "category=Wildlife Safari" \
  -F "sub_category=Luxury Safari"
```

### Response

Redirects to tours list with success message: "You Updated a Tour."

## Publish/Unpublish Tour (Admin)

<api method="PUT" path="/addtour/publish/{id}" />

Toggle the publication status of a tour (publish or unpublish).

### Authentication

<Note>
  Requires authentication and verified email
</Note>

**Middleware:** `auth`, `verified`

### Path Parameters

<ParamField path="id" type="integer" required>
  The unique tour ID
</ParamField>

### Behavior

- If `published = 0`, sets to `1` (publish)
- If `published = 1`, sets to `0` (unpublish)

### Example Request

```bash
curl -X PUT https://yourdomain.com/addtour/publish/123 \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "X-CSRF-TOKEN: YOUR_CSRF_TOKEN"
```

### Response

Redirects to tours list with updated publication status.
