The search functionality on Okulink Safaris helps you quickly find specific tours based on your preferences. This guide explains how to use the search feature effectively.

## Accessing Search

The search feature is available throughout the site:

<Steps>
  <Step title="Navigate to Search">
    Access the search page at `/search` or use the search functionality in the navigation menu
  </Step>
  
  <Step title="Enter Search Term">
    Type your search query into the search box
  </Step>
  
  <Step title="View Results">
    Search results are displayed automatically, showing matching tours
  </Step>
</Steps>

<Info>
The search route is accessible at `/search` and uses the HomeController's search method to process queries.
</Info>

## How Search Works

The Okulink Safaris search system looks for matches in tour information.

### Search Parameters

When you enter a search term, the system searches through:

- **Hotel/Accommodation Names**: Finds tours based on where you'll stay
- **Destination Information**: Matches against tour destinations

<Note>
The search uses a LIKE query pattern, meaning it will find partial matches. For example, searching for "Serengeti" will find "Serengeti National Park" and "Central Serengeti Lodge".
</Note>

### Search Query Format

The search accepts a parameter called `item` that contains your search term:

```
/search?item=your-search-term
```

<Tip>
Use specific terms related to hotels, accommodations, or destinations for best results.
</Tip>

## Search Limitations

Understand these search characteristics for best results:

### What's Searchable

Currently, the search focuses on:
- Hotel names
- Accommodation names
- Tours with non-null hotel values

### What's Not Searchable

The following are not directly searchable through the main search:
- Tour prices
- Meal inclusions
- Transport types
- Tour categories (use category browsing instead)
- Date ranges

<Tip>
For category-based browsing, use the category and subcategory navigation described in the [Browsing Tours](/user-guide/browsing-tours) guide.
</Tip>

## Search Tips and Best Practices

### Getting Better Results

<Steps>
  <Step title="Be Specific">
    Use specific hotel or lodge names when you know them (e.g., "Serengeti Sopa Lodge" instead of just "lodge")
  </Step>
  
  <Step title="Try Different Terms">
    If your first search doesn't yield results, try variations:
    - Different spellings
    - Shorter terms
    - Related keywords
  </Step>
  
  <Step title="Use Partial Matches">
    The search finds partial matches, so you don't need to type the complete name
  </Step>
  
  <Step title="Check Suggestions">
    Review the random tour suggestions for alternatives you might not have considered
  </Step>
</Steps>

## Alternative Finding Methods

If search doesn't give you the results you're looking for, try these alternative methods:

### Browse by Category

Navigate to `/tours/{category}` to see all tours in a specific category.

<Info>
See the [Browsing Tours](/user-guide/browsing-tours) guide for detailed information on category navigation.
</Info>

### Browse by Subcategory

Use `/tours/categories/{sub_category}` for more refined filtering.

### View All Safaris

Visit `/safaris` to browse the complete tour catalog.

## Pagination

Search results are paginated for better performance and usability.

<Steps>
  <Step title="View Initial Results">
    The first page shows up to 10 matching tours
  </Step>
  
  <Step title="Navigate Pages">
    Use the pagination controls at the bottom to view additional results
  </Step>
  
  <Step title="Page Numbers">
    The current page number and total pages are displayed
  </Step>
</Steps>

<Note>
Pagination starts at page 1, with 5 items indexed per page in the results display.
</Note>
